Recovery agent patterns

The product starts with what is live, then expands carefully into future supported patterns.

For restaurants

SaanaOS is the first live restaurant use case. Missed Call Recovery is live and turns missed calls into direct ordering opportunities.

For agencies

Package recovery outcomes for clients without forcing every workflow into a single SaaS dashboard.

For developers

Send events, inspect runs, receive outcomes, and integrate with existing systems through APIs and delivery channels.

For future platforms

Post-Checkout Revenue and Post-Checkout Growth are future supported patterns, not live integrations yet. View the Post-Checkout draft or open the calculator.

Delivery and traceability

Webhooks are for realtime delivery. SFTP pull is for batch delivery. All timestamps are UTC.

RealtimeWebhooks deliver outcomes and action updates as they happen.
BatchSFTP pull provides UTC day-window files for client systems.
TraceEvent → Attempts Engine → Agent → Outcome → Webhook / SFTP.
