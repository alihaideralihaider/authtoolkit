Security

Security is important to us. AuthToolkit welcomes responsible security research and vulnerability reports that help improve the safety of our platform.

Reporting a vulnerability

If you believe you have discovered a security vulnerability affecting AuthToolkit, please report it responsibly.

Send details to:

security@authtoolkit.com

Please include:

  • A clear description of the vulnerability
  • Steps required to reproduce the issue
  • Any proof-of-concept code or screenshots
  • The potential impact of the vulnerability

Responsible disclosure

We request that researchers provide reasonable time for investigation and resolution before publicly disclosing vulnerabilities.

AuthToolkit will make reasonable efforts to acknowledge valid reports and address confirmed security issues.

Scope

This policy applies only to vulnerabilities affecting the AuthToolkit service itself. Reports concerning third-party services or external infrastructure providers should be directed to those providers.

Prohibited actions

Security testing must not:

  • Disrupt service availability
  • Access data that does not belong to you
  • Attempt large-scale automated attacks
  • Exploit vulnerabilities beyond proof-of-concept testing

Contact

Security reports should be sent to:

security@authtoolkit.com
